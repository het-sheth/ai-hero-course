## Database Migrations (01.04)

- [[before-we-start/database-migrations|Database Migrations]] — SQLite is a file (`data.db`); schema.ts → `db:generate` → `db:migrate`; reset by deleting `data.db` + `db:seed`

> [!NOTE] Takeaway worth remembering
> Agents reliably run `db:generate` after a schema edit but often skip `db:migrate` — so when I let an agent change the schema, I need to check the migration actually got applied.
