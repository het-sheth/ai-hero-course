## Build A Feature (03.04)

- [Build A Feature](/day-1-fundamentals/build-a-feature.md) — first feature: a 1–5 star course-rating system (no written reviews), shown on the list page and course page. The point wasn't the feature but the **workflow**: lightweight 1–2 sentence prompt, then observe Claude's default behavior (Explore subagent? what files? what questions?), review its plan and steer before implementing, flip to accept-all-edits (shift+tab), and stay context-paranoid — `/context` repeatedly, nervous around the **40% (~80K token)** dumb-zone line.
- Testing covered the enrolled-student widget (click star → toast + average updates, re-click = upsert), the `/courses` card display, and the non-enrolled / instructor cases.

- **Did the exercise myself** — built the star-rating feature in my cohort fork: [het-sheth/cohort-004-project#1](https://github.com/het-sheth/cohort-004-project/pull/1) (`course_ratings` table + upsert, rating service, detail-page widget with toast, average on list cards). Verified with typecheck + build + a rolled-back DB smoke test. The Solution walkthrough is now captured on the page too.

## Non-Determinism (03.05)

- [Non-Determinism](/day-1-fundamentals/non-determinism.md) — a detour setting expectations: agents are **next-token-chooser machines**, so the same prompt on the same codebase can give different answers. Responses sit on a probability curve (most sensible, occasional weird outliers). It's baked in, not a setup bug — "ride the wave," and you can make agents more consistent later via AFK feedback loops.

## Showing Context in the Status Line (03.06)

- [Showing Context in the Status Line](/day-1-fundamentals/showing-context-in-status-line.md) — Claude Code hides context usage by default; surface it in the status line (lesson uses **ccstatusline**) so the dumb-zone line is always one glance away. Post-1M-window, show **tokens then percentage** (`186.2k (17.3%)`); worry around 80–100k tokens.
- Applied to my own setup: had Claude edit `~/.claude/statusline-command.sh` so ctx shows tokens + % and reordered to `model · ctx · 5h · 7d · branch · workdir`.

## Why Plan Mode Sucks (03.07)

- [Why Plan Mode Sucks](/day-1-fundamentals/why-plan-mode-sucks.md) — plan mode (shift-tab) drops file-writing so the agent plans first, but the **interview phase truncates** → wall-of-text plan you can't tell you're aligned with. Brooks' **"shared design concept"** is what's missing. Better: **explore → interview hard → implement**, via Matt's **`/grill-me`** skill (starts almost every session).
