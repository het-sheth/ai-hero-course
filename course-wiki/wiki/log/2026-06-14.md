## The Grill → Execute → Clear Loop (03.08)

- [Grill → Execute → Clear](/day-1-fundamentals/grill-execute-clear.md) — the working loop that
  replaces [plan mode](/day-1-fundamentals/why-plan-mode-sucks.md): **grill** (interview hard with
  `/grill-me` to reach a shared design concept), **execute** (implement against it), **clear** (wipe
  context for the next task).
- The exercise: design a **lesson comments feature**, deliberately left vague. The interesting work
  is resolving the interconnected design questions — can instructors comment / moderate / delete or
  hide? are comments enrolled-only or public? can students see each other's? — before any code.
- The `/grill-me` skill is tiny: *interview me relentlessly… walk down each branch of the design
  tree, resolving dependencies one by one… recommended answer per question… one at a time… and if a
  question can be answered by exploring the codebase, explore instead.* The **"explore" verb** keeps
  it from asking what it could check itself; the **recommended answers** keep it fast.
- Run it in **default mode** (not plan, not auto; accept-edits off), at `.claude/skills/grill-me/SKILL.md`.
  Take notes on *how* it designs and how you'd improve the skill, continue until you both agree
  ("ready to implement, let's go"), then implement and check the result matches the understanding.
- Matt's solution (commit `03.08.02`) shipped lesson comments **with soft-delete and moderation**.

## Compaction (03.09)

- [Compaction](/day-1-fundamentals/compaction.md) — the "open the lid" lesson on what happens when a
  session runs through the [dumb zone](/day-1-fundamentals/constraints-of-llms.md) to the context
  limit. Claude Code reserves an **auto-compact buffer** (~33k tokens / 16.5% at the end of
  `/context`); crossing into it triggers **compact**, which has an LLM summarize the conversation
  into a small markdown form (in the demo: 49% → 12% / ~23k tokens, full transcript saved to a
  referenced file). You can pass **custom summarization instructions** ("I've just implemented a
  feature and want to QA it") to bias what's kept.
- Key opinion: **prefer clean context over compaction.** Repeated compaction leaves **"sediment"** —
  layered summaries that make output less predictable. Compacting multiple times = anti-pattern.
  Matt compacts rarely (≤ once/convo), only on hard long-running tasks where he's working *with* the
  LLM directly and wants to stay in the smart zone. The course is heading toward autonomous,
  clean-context workflows where you don't sit there triggering compaction.

## Handing Off (03.10)

- [Handing Off](/day-1-fundamentals/handing-off.md) — the answer to compaction's open problem: you're
  mid-session ("blue") and discover a side task ("yellow", e.g. a broken test). Don't do it inline
  (might burn blue's budget) and don't `/clear` (you still need blue). Instead **give yellow its own
  context window** by handing off.
- The `/handoff` skill (cohort commit `03.10.01`, real file fetched) writes a temp markdown doc
  (`mktemp -t handoff-XXXXXX.md`) summarising the convo so a fresh agent can continue. Key behaviours:
  **suggests the next session's skills** (a `/grill-me` handoff says keep grilling), **doesn't
  duplicate** other artifacts (links PRDs/plans/diffs by path/URL), and **tailors to the next
  session** via its `argument-hint`. It's compaction's *output* without the sediment.
- Two patterns: spin off a bug/test fix into a clean window; or **expand→contract** in planning —
  hand off from a long [/grill-me](/day-1-fundamentals/grill-execute-clear.md) to a focused
  research/prototype session, then hand back a condensed doc. Documents shuttle between agents so no
  one window holds everything.

## Day 2 · Steering begins — What Is An AGENTS.md File (04.01)

- [What Is An AGENTS.md File](/day-2-steering/agents-md-files.md) — first steering mechanism: a
  git-checked-in markdown file (open standard: Gemini CLI, Devin, Codex, Cursor) that customises agent
  behaviour, sitting just below the system prompt. **Claude Code ignores AGENTS.md and reads
  CLAUDE.md.** Three gotchas: it's **global** (every conversation in the repo), **costs tokens every
  request** (the 2000-line pirate demo burned ~10% of the window), and is **often ignored by design**
  (Claude Code injects "may or may not be relevant… don't respond unless highly relevant").
- Avoid bloat: the **ball-of-mud** feedback loop, the **instruction budget** (~150–200 instructions
  for frontier thinking models, per Humanlayer/Kyle), and **staleness** (never document file paths —
  describe capabilities). Don't run `claude init` (auto-generated, stale, discoverable). Minimum
  viable CLAUDE.md: one-line project description + package manager (if not npm) + non-standard build
  commands. Everything else → [progressive disclosure](/day-2-steering/progressive-disclosure.md) /
  separate files / [skills](/day-2-steering/agent-skills.md).
- **Applied to my own setup:** audited my ~150 CLAUDE.md/AGENTS.md files against these principles
  (see the separate audit) — found a duplicate global, a 681-line outlier, and a cluster of
  byte-identical 238-line templated files across rad-enclave. Cohort solution ref: `04.01.01`.
