- [Prompting in the Terminal](/gtk-claude-code/prompting-in-the-terminal.md) (02.03) — `@` to reference files into context, `Ctrl+S` to stash a prompt (and `Ctrl+C` to drop it), pasting images.

- [Running Bash Commands](/gtk-claude-code/running-bash-commands.md) (02.06) — `!` bash mode (output into Claude's context), `Ctrl+B` to background a dev server, `Ctrl+Z`/`fg` to suspend Claude and hide commands from it.
- [Permissions](/gtk-claude-code/permissions.md) (02.07) — strict-by-default approve/deny model, `.claude/settings.local.json` allow/deny + wildcards, sharing via `settings.json`, and web-fetch permissions.

## Day 1 · Fundamentals (started)

Opened Module 03 with the two foundational lessons:

- [The Constraints of LLMs](/day-1-fundamentals/constraints-of-llms.md) (03.01) — the four constraints: smart zone vs dumb zone (attention scales quadratically; smart zone ≈ first 80–100k tokens regardless of the 1M window), the fuzzy-JPEG database (why hallucinations happen), statelessness (every clear wipes memory → codebase quality matters), and knowledge cutoff.
- [Subagents](/day-1-fundamentals/subagents.md) (03.02) — context-saving delegation: the orchestrator spawns a fresh-context subagent for focused work (e.g. exploration, sometimes on a cheaper model) and gets back just a summary, keeping the main window clean for implementation.
- [Codebase Exploration](/day-1-fundamentals/codebase-exploration.md) (03.03) — exploration is foundational (LLMs are stateless); a generic prompt does a shallow ~6-file skim, but putting the word **"explore"** in the prompt triggers a dedicated Explore subagent that aggressively reads files and returns a rich summary. Covers both the Problem (exercise) and Solution (walkthrough) sub-pages.
