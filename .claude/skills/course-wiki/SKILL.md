A living second-brain wiki for Matt Pocock's *AI Coding for Real Engineers* cohort, written
**Markdown-canonical** (Karpathy LLM-wiki pattern): atomic Markdown concept pages are the source
of truth; a deterministic build regenerates the phosphor-terminal **HTML view** in `site/`.

**Wiki root:** `~/projects/ai-hero-course/course-wiki/`
(This skill is project-scoped: it lives only in this repo at `.claude/skills/course-wiki/`
and loads automatically when you work inside `~/projects/ai-hero-course/`.)

## The one inviolable rule: never invent content
Pages capture **Het's** learning. Prose comes ONLY from material he provides ‚Äî lesson
text/transcripts, his notes, or what he tells you. Do NOT fill a page from general knowledge.
Missing material ‚Üí leave a `status: stub` page and ask him. Mark anything uncertain `{{unverified}}`.

## Layout
- `raw/<module>/` ‚Äî immutable source material you may be given. Read, never rewrite.
- `wiki/<module>/<slug>.md` ‚Äî atomic concept pages (one concept per file).
- `wiki/log/<date>.md` ‚Äî dated entries.
- `site/` ‚Äî GENERATED by `build.mjs`. Never hand-edit.
- Schema + body conventions: see `course-wiki/CONVENTIONS.md`. Entry point: `course-wiki/START_HERE.md`.

## Common intents
**"Add/Write the concept page for X"** (Het shared material)
1. Optionally drop the source into `raw/<module>/`.
2. Create `wiki/<module>/<slug>.md` with frontmatter (`type` required; plus `title`,
   `description`, `status`, `sources`) ‚Äî fill the body from his material ONLY.
3. Add `related:` slugs that resolve; flip the matching hub stub into a real page by changing its
   `status` and writing the body.
4. `npm run check` ‚Üí fix broken links.

**"Log today"** ‚Üí create `wiki/log/<date>.md` with `type: log`/`title`/`description`, transcribe
his notes, `[Label](/module/slug.md)`-link the concepts it touched.

**"Update concept X"** ‚Üí edit `wiki/<module>/<slug>.md`; refresh `related:`/`sources:`.

## Build & publish
- `npm run check` (validate links) ‚Üí `npm run build` (regenerate `site/`) ‚Üí open `site/index.html`.
- `main` is PR-protected. Branch + PR for every change:
  `git checkout -b wiki/<topic>` ‚Üí commit ‚Üí `gh pr create --fill` (GH_HOST=github.com) ‚Üí
  `gh pr merge --squash`. CI rebuilds + redeploys Pages on merge. **Never push to `main`.**
